# Contributing

Docs-only corrections are welcome when they improve clarity, fix broken public links, or update stale public wording.

## Good Contributions

- Typo fixes.
- Broken public link fixes.
- Clearer public usage wording.
- Corrections that match the live product surface.

## Do Not Include

- Private source code or private repository structure.
- Secrets, tokens, environment variables, or internal URLs.
- Customer, client, patient, booking, transcript, portfolio, payment, or personal data.
- Proprietary implementation details or non-public roadmap claims.
